Introduction

Mesa 7.6 includes implementations of the OpenGL ES 1.1 and OpenGL ES 2.0 APIs.

Basically, the ES 1.1 and 2.0 libraries are Gallium state trackers which are subsets of Mesa. Any Gallium driver that works with Mesa should work with the ES 1.1 and 2.0 libraries.

Source Code

The source code for the ES front-ends is in the src/gallium/state_trackers/es/ subdirectory.

The ES1 and ES2 libraries share a subset of the regular Mesa sources. A symlink mirror of the src/mesa/ tree is created under src/gallium/state_trackers/es/mesa/. Some Mesa sources which aren't used for ES (ex: display list code) are omitted and a few other files are replaced with ES-specific version (ex: mfeatures.h). Also note that because of the #ifdef FEATURE_x checks in Mesa, the .o files produced in the ES tree will be different from the .o files in the main Mesa tree. That's why a separate symlinked directory tree is used.

Much of the ES1 and ES2-specific code is auto-generated with Python scripts. See the src/gallium/state_trackers/es/*.py files.

Source code specific to ES1 is in src/gallium/state_trackers/es/es1/.
Source code specific to ES2 is in src/gallium/state_trackers/es/es2/.
Source code common to ES1 and ES2 is in src/gallium/state_trackers/es/es-common/.

OpenGL ES 1.1

The OpenGL ES 1.1 header files are found in include/GLES/

The OpenGL ES 1.1 (common profile) library name is libGLESv1_CM.so.1.1.0

Example programs can be found under progs/es/es1.

OpenGL ES 2.0

The OpenGL ES 2.0 header files are found in include/GLES2/

The OpenGL ES 2.0 library name is libGLESv2.so.2.0.0

Example programs can be found under progs/es/es2.

EGL

The EGL header files are found in include/EGL/

The EGL library name is libEGL.so.1.0
